## Question

In 3-4 sentences: how should I use vbt.Splitter.from_expanding with vbt.Param for walk-forward parameter optimization? Cite vbt function names.

## Answer

Use `vbt.Splitter.from_expanding` to create the walk-forward splits, then pass that splitter into your pipeline via `vbt.cv_split` or `splitter.apply`. Combine it with `vbt.Param` to run the parameter grid on each training split, and use a `selection` function in `vbt.cv_split` to pick the best parameter combination per split. For example, `@vbt.cv_split(splitter=..., splitter_kwargs=..., selection=...)` lets you optimize on the train set and validate on the corresponding test set. See the cross-validation tutorial and the `cv_split`/`split` examples: https://vectorbt.pro/pvt_16ebf9ef/tutorials/cross-validation/
